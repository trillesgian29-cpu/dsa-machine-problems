## 🏭 Case Study: Defective Machine Simulation

### **Problem Statement**

In the study of Data Structures and Algorithms (DSA), machine problems are designed to bridge theory with practical application. One such problem is the **Defective Machine Simulation**, where a factory of machines produces items, each machine identified by a unique ID. 

Under normal conditions, every machine produces an even number of items, meaning its ID should appear **twice** in the dataset. However, one faulty machine produces an odd number of items, causing its ID to appear only **once**. The challenge is to detect this defective machine efficiently.

### **Solution Approach**

To solve this problem, we applied fundamental DSA concepts:

- **Arrays** — Store machine IDs in a structured dataset
- **Sorting Algorithms** — Organize IDs (bubble, insertion, selection, merge, quick) so duplicates can be easily compared
- **Searching Techniques** — Locate and verify machine IDs quickly (linear and binary search)
- **Linked Lists** — Provide dynamic representation useful for frequent insertions/deletions

### **Real-World Application**

This approach reflects how these algorithms are applied in real-world problem solving. In manufacturing systems, **anomaly detection** is critical:

- Logs of machine outputs are stored in arrays or databases
- Data is sorted for analysis
- Irregularities are searched and identified
- Dynamic data is managed with linked structures

By simulating the faulty machine scenario, we demonstrate how core DSA techniques can be leveraged to **detect errors, improve reliability, and ensure quality control** in real industrial environments.
